# 量化交易大模型Agent系统 - 简历项目描述（研究型岗位版）

## 项目名称
**基于大模型的量化交易Agent系统**  
*LLM-powered Quantitative Trading Agent System*  
*2024.01 - 2024.12 | 个人研究项目 | [GitHub](https://github.com/Torrieee/quant_trading_agent)*

---

## 项目概述（研究定位）

设计并实现了一个融合大模型能力的智能交易Agent系统，采用经典的Agent架构（感知-决策-执行-评估循环），重点研究并实践了大模型在金融交易决策中的应用。系统集成了Function Calling、RAG（检索增强生成）和LLM驱动的策略生成等前沿技术，实现了从自然语言理解、智能决策到策略执行的端到端智能化流程。项目在2018-2025年7年历史数据上完成验证，通过参数优化实现年化收益率16.45%，夏普比率0.91。

---

## 核心研究贡献

### 1. 大模型Agent架构设计与实现 ⭐⭐⭐

**研究问题**：如何将大语言模型的自然语言理解和生成能力有效集成到金融交易决策流程中？

**技术实现**：
- **Function Calling框架**：设计了符合OpenAI规范的Function Calling接口，定义了5个交易相关的工具函数（获取市场数据、分析市场状态、策略推荐、仓位计算、回测执行），实现了LLM自动解析用户意图并调用相应函数的能力
- **工具函数设计**：实现了参数验证、错误处理和结果标准化，确保LLM调用的可靠性和准确性
- **验证结果**：成功验证LLM自动获取AAPL市场数据、识别上升趋势（ADX 63.93）、推荐momentum策略、计算凯利仓位（8.33%）等功能，证明了Function Calling在交易场景中的有效性

**研究价值**：
- 探索了大模型在金融领域的工具调用范式
- 验证了自然语言到交易决策的端到端流程
- 为金融大模型Agent提供了可复现的实现框架

---

### 2. RAG系统在交易决策中的应用研究 ⭐⭐⭐

**研究问题**：如何利用历史交易经验增强大模型的决策能力，减少"幻觉"并提高决策准确性？

**技术实现**：
- **向量数据库构建**：基于FAISS实现高效的向量存储和检索，使用OpenAI Embeddings对历史交易记录进行向量化
- **知识库构建**：设计了文档分块策略，将交易记录（标的、策略、市场状态、收益结果）转换为结构化文档
- **检索策略**：实现了基于语义相似度的检索算法，支持k-NN搜索和历史经验匹配

**应用场景**：
- 根据市场状态检索相似历史策略表现
- 基于历史经验提供交易建议，减少LLM的"幻觉"
- 实现经验知识的积累和复用机制

**研究价值**：
- 验证了RAG在金融决策支持中的有效性
- 探索了历史经验如何增强LLM的专业知识
- 为金融领域的知识增强型Agent提供了实践案例

---

### 3. LLM驱动的策略生成与优化研究 ⭐⭐⭐

**研究问题**：如何利用大模型的自然语言理解和生成能力，实现从策略描述到可执行策略的自动转换？

**核心技术**：
- **自然语言策略生成**：实现了从自然语言描述（如"当价格低于20日均线时买入"）到完整策略配置的自动生成
- **策略解释与分析**：使用LLM自动生成策略逻辑说明和回测表现分析，提高策略可解释性
- **智能策略组合推荐**：基于市场状态和风险偏好，LLM自动推荐策略组合及权重分配
- **策略改进建议**：LLM基于当前策略表现和市场环境，提供参数调优和策略调整建议

**技术创新**：
- 首次将LLM策略生成能力应用于量化交易领域
- 实现了从"想法"到"策略"的快速原型验证
- 探索了大模型在策略设计和优化中的角色

**研究价值**：
- 降低了策略开发的技术门槛
- 提高了策略设计的迭代效率
- 为AI驱动的策略研发提供了新范式

---

### 4. 多模态Agent决策框架 ⭐⭐

**研究问题**：如何将结构化数据（市场数据、技术指标）与LLM的自然语言能力有机结合？

**技术实现**：
- **数据融合**：实现了结构化市场数据与LLM文本理解的融合机制
- **状态表示**：设计了30+维特征向量（价格、技术指标、市场状态）作为Agent状态空间
- **决策流程**：市场状态识别（基于ATR、ADX等指标）→ LLM策略推荐 → 执行决策 → 性能评估

**验证结果**：
- 在2018-2025年7年数据上回测，系统准确识别上升趋势（ADX 63.93）并推荐momentum策略
- 通过参数优化将策略年化收益率从6.32%提升至16.45%（提升2.6倍）
- 交易胜率达到72.31%，验证了多模态决策框架的有效性

## 面试准备（研究相关）

### Q1: 为什么选择将大模型应用到量化交易场景？
**A**: 量化交易决策涉及大量非结构化信息和复杂推理，传统方法难以处理。大模型具有强大的自然语言理解和生成能力，可以帮助：
1. 理解自然语言策略描述，降低开发门槛
2. 解释策略逻辑和决策过程，提高可解释性
3. 结合历史经验（RAG）提供更准确的建议
4. 实现端到端的智能化流程

### Q2: RAG系统如何减少LLM的"幻觉"？
**A**: 传统LLM依赖预训练知识，可能产生"幻觉"。RAG系统通过：
1. 从历史交易记录中检索相似的真实案例
2. 将检索到的经验作为上下文提供给LLM
3. LLM基于真实历史经验而非仅依赖预训练知识生成建议
4. 验证结果表明，基于历史经验的建议更准确可靠

### Q3: Function Calling在金融场景中的优势是什么？
**A**: Function Calling实现了自然语言到工具调用的无缝衔接：
1. 用户可以用自然语言描述需求（如"分析AAPL的市场状态"）
2. LLM自动解析意图并调用相应工具函数
3. 工具函数返回结构化数据，LLM整合后生成回答
4. 既利用了LLM的语言理解能力，又保证了数据获取的准确性

### Q4: 这个项目的研究价值是什么？
**A**: 本项目的研究价值在于：
1. **范式探索**：首次系统性地将大模型Agent技术应用于量化交易，探索了新的研究范式
2. **技术创新**：实现了Function Calling和RAG在金融场景的实践，验证了技术可行性
3. **可复现性**：提供了完整的代码框架和实验验证，为后续研究提供基础
4. **实用价值**：降低了策略开发门槛，提高了决策效率
